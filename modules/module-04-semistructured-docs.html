DU Service Platform in Automation Cloud

The Document Understanding service in UiPath Automation Cloud is your central hub for building, managing, and deploying AI extraction models. It eliminates the need for separate AI Center configuration for most use cases.

Accessing Document Understanding

  1. Log into cloud.uipath.com
  2. Select your tenant
  3. Navigate to Document Understanding service in the left menu
  4. You will see your projects — Classic and Modern

Classic Projects — What You Should Know

Classic DU projects are the traditional framework for UiPath Document Understanding. They offer compatibility with OCR engines and rely on AI Center for model hosting.

  • Cost-effective for OCR-heavy tasks — free OCR usage that does not consume AI units; first 25 pages per document are processed free.
  • Established capabilities — well-suited for regex-based extraction and Intelligent Keyword Classifier (IKC) classification.
  • Broader infrastructure support — works on Automation Cloud, Suite, and Standalone deployments.
⚠️ Important
Important: UiPath recommends creating all new projects using Modern Projects. Classic activities remain available and supported, but are no longer being enhanced. Transitioning to Modern is strongly encouraged for access to active learning, generative AI pre-annotation, and ongoing feature updates.
💡 Tip
Recommendation: Use Modern Projects for new implementations on Automation Cloud. They offer faster model development (80% faster), no AI Center setup, and guided training experience.

Modern Projects

Modern Projects is the next-generation AI-powered model training experience within UiPath Document Understanding. It uses an iterative process between annotators and an ML model to minimize the amount of labeled data needed.

Key Benefits of Active Learning

  • 🚀 80% faster model training — from a week down to a day
  • 👥 No ML expertise required — anyone can train AI models
  • 🤝 Human + AI collaboration — guidance on model optimization
  • 📊 Built-in performance analytics — instant model evaluation
  • 🔄 Background training — model improves as you annotate

Modern Experience: Build → Measure → Publish → Monitor

🔨 Build

  • Upload & classify your documents (automatic AI classification)
  • Generative AI pre-annotates fields — you validate/correct
  • Follow recommendations on how many samples to add
  • Model trains in the background as you annotate
  • Minimum recommended: 30 documents per document type

📊 Measure

  • Evaluate model readiness with performance metrics (F1, precision, recall per field)
  • Field-level performance shows which fields need more attention
  • Distribution of annotated vs unannotated documents
  • Guided recommendations on where to improve

🚀 Publish

  • Choose the model version to deploy
  • One-click deployment — model becomes available as a service
  • Easy versioning of your entire project
  • Consume from Studio, Studio Web, or via APIs

👁 Monitor

  • Insights dashboard shows live automation performance
  • Drill down into individual document processing status
  • Visibility into human-in-the-loop events and corrections
  • Search through historical document processing records

UiPath Helix — Classifier & Extractor

UiPath Helix is UiPath's next-generation LLM-based AI framework for Document Understanding. It includes two distinct models: the Helix Classifier (for identifying document types) and the Helix Extractor (for extracting field data). Both are available in Modern Projects and significantly reduce the annotation effort needed to build high-accuracy automations.

Helix Classifier

The Helix Classifier is an LLM-based document classification model that automatically identifies document types with minimal training data.

Key Characteristics

  • Classifies documents into defined document types using generative AI
  • Requires significantly fewer training samples compared to the Intelligent Keyword Classifier
  • Works within Modern Projects — no separate AI Center configuration needed
  • Background model training as you annotate and validate classification results
  • Supports splitting of multi-document PDFs into individual document types

Availability

  • Available in Modern Projects on Automation Cloud
  • US-based tenants (excluding GxP and Government Cloud)

Helix Extractor

The Helix Extractor 1.0 is an LLM-based data extraction model built into Document Understanding. It leverages a wide variety of training documents to handle common document types with minimal annotation requirements.

Key Characteristics

  • 🎯 ~15% fewer false positives and ~17% fewer false negatives compared to previous models
  • ✏️ Reduced annotation effort — only one annotation per document required, not per field instance
  • 📊 Better confidence scores — stronger correlation between confidence values and actual extraction accuracy
  • Text matching only — does not support summarisation or other types of text analysis
  • Latin script languages only (non-Latin scripts not supported in v1.0)
  • Field names must use natural language and proper grammar for optimal performance

Supported Document Types

  • ✅ Invoices, Receipts, Purchase Orders, and semi-structured documents
  • ❌ Not supported: Financial Statements; Invoices (China, Hebrew, Japan variants)

Availability

  • Modern Projects in US-based tenants (excluding GxP and Government Cloud)
  • Public endpoints in Europe and Japan for specific invoice and receipt types
  • Helix Extractor 2.0 (Public Preview) — adds multimodal processing, non-Latin script support, and improved table handling. Targeting GA mid-2026.

How to Activate Helix Extractor for Existing Document Types

  1. Delete all fields from your sample documents in the project.
  2. Re-annotate and confirm those fields.
  3. Repeat until training initiates.
  4. Check the model version number — version 24.7 or above confirms Helix Extractor is active.

For full details, refer to the UiPath Helix Extractor 1.0 documentation → and Helix Extractor 2.0 (Public Preview) →

Annotation in Modern Experience

Annotation is the process of labeling document examples to tell the model what to extract. In Modern Experience, annotation is guided and accelerated by Generative AI pre-annotation.

  1. Upload documents to your Modern project (minimum 30 per document type)
  2. Automatic classification: AI assigns document types; review and correct any mistakes
  3. AI pre-annotation: Generative AI suggests field values for each document
  4. Validate predictions: Review each suggestion — accept if correct, correct if wrong
  5. Background training: Model retrains automatically as you validate
  6. Check recommendations: Platform guides you on which documents to annotate next
📌 Note
Annotation Best Practices: Annotate a diverse set of document samples covering different vendors, layouts, and edge cases. The more diversity in your training set, the more robust your model will be.

Migrating a Project from Dev to Production

When your model is ready, you can migrate it from your development environment to production by exporting the annotated dataset and importing it into the target project. UiPath provides a structured process for this within Modern Projects.

Step 1: Export Data

  1. Open your project and navigate to the Publish section.
  2. Go to the Export tab.
  3. Select Export data (ensure a project version exists first).
  4. Choose the desired document type from the dropdown menu.
  5. Select the project version to export from its dropdown.
  6. Click Export to initiate the process.
  7. When ready, a popup appears with a download option. Download immediately or retrieve it later via the three-dot menu.
📌 Note
Only annotated and confirmed items (marked with a green shield icon) are exported. Datasets from project versions created prior to May 2024 (GA date) are not supported for export.

Step 2: Import Datasets

  1. Open the target project (e.g. your production environment).
  2. Select Add document type to create a new custom document type.
  3. In the newly created document type, select Upload.
  4. Choose the exported .zip file and wait for the upload to complete.
  5. Once the upload finishes, training starts automatically.
📌 Note
You can only import zip datasets directly into a document type — not at the project level.
